## 六、开发规范（图片显示）

**图片统一走接口**：`/exam/api/getImage?fileName=文件路径`，请求需带 token（请求头 `Authorization` 或 URL 参数 `token`）。

| 场景 | 使用方式 |
|------|----------|
| 需请求头带 token 的展示（题目/试卷/成绩等） | `fetchImageAsBlobUrl(path)` 或 `replaceImgPlaceholdersWithAuth`；单图用组件 `<AuthImage :path="img.url" />` |
| 题中图（content 中 `[img]` 占位符） | `parseContentWithImages(content, questionContentImages)` 得到 `part.path`，用 `<AuthImage :path="part.path" />` |
| 题下图 / 答案图（attachedImage、answerImage） | `<AuthImage :path="img.url" />` 或上述异步替换 |

**格式**：接口可能返回 jpg/png/gif/bmp/webp 或 **SVG**，前端 `fetchImageAsBlobUrl` 已按路径/Content-Type 做 SVG 处理，可直接显示。

详见 [中车2.0前端开发规则.md](./中车2.0前端开发规则.md) 之「图片显示规范」。

## 七、开发规范（试卷用途/规则类型 ruleType）

**凡涉及「职业认定、竞赛、模拟考核」选择的模块，必须使用字段 `ruleType`，取值为数值 0/1/2。**

| ruleType | 含义     | 界面显示 |
|----------|----------|----------|
| 0        | 职业认定 | 职业认定 |
| 1        | 竞赛     | 竞赛     |
| 2        | 模拟考核 | 模拟考核 |

**涉及页面/组件**：
- 组卷管理：`PaperManagement2.vue`（列表展示 formatRuleType）、`AutoPaperModal.vue`、`ManualPaperPage.vue`
- 组卷规则：`QuestionGroupRule.vue`、`RuleDialog.vue`

**约定**：下拉使用 `v-model.number` 绑定 0/1/2；接口传参使用 `ruleType`；列表/详情展示用统一 `formatRuleType` 格式化。后续新增模块凡包含该选择，须参考本规范。

详见 [中车2.0前端开发规则.md](./中车2.0前端开发规则.md) 之「试卷用途/规则类型（ruleType）统一规范」。
