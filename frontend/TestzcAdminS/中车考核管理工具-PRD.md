# 中车考核管理工具设计文档

## 一、项目概述

### 1.1 项目名称
中车考核管理工具（NewZCAdmin）

### 1.2 项目背景
- **业务背景**：中车集团职业技能考核、职业认定、竞赛及模拟考核的管理平台，需支撑考核业务全流程（题库、组卷、考试、成绩、证书等），传统人工管理方式已无法满足高效、准确、可追溯的管理需求。
- **技术背景**：Vue 3 + Vite + Element Plus 前端；组件切换式页面导航（activeView 非 SPA 路由）；JWT Token 认证；需与后端 API 对接。
- **市场背景**：考核业务规范化和信息化管理需求提升，需多角色协同、流程可追溯。

### 1.3 项目目标
- **主要目标**：实现考核申请→组卷→审卷→派发→阅卷→成绩→证书全流程自动化管理，支持职业认定、竞赛、模拟考核等类型。
- **次要目标**：多角色权限控制、操作日志、数据导入导出、线上线下考试、成绩分析。
- **可量化指标**：系统响应时间小于 2 秒，支持主流浏览器（Chrome/Edge 最新版），首屏加载小于 3 秒。

### 1.4 术语说明
| 术语 | 说明 |
|------|------|
| ruleType | 试卷用途/规则类型：0=职业认定、1=竞赛、2=模拟考核，组卷管理、组卷规则、资源题库中统一使用数值 |
| passScore | 组卷规则达标分数字段 |
| activeView | 主框架 Main.vue 当前显示的视图标识，用于组件切换 |
| 试卷申请 | 考核站管理员提交的试卷申请，包含上传的附件（考试列表文件，支持 xlsx、doc、docx、pdf） |
| 派发记录 | 管理员组卷并审核通过后，将试卷派发给考核站的记录，需关联对应试卷申请 |
| 加密试卷包 | 后端将多张试卷生成 A3/A4 Word 文件后打包成的加密压缩包（如已实现） |
| 解压密码 | 用于解密试卷包的密码，密文存储，仅可查看一次明文（如已实现） |

### 1.5 全局约束规则

| 约束规则 | 说明 | 适用场景 |
|---------|------|----------|
| 职工编号来源约束 | 考试人员涉及的职工编号须来源于职工列表，不存在时禁止添加或导入并给出明确提示 | 考试人员新增、导入、成绩录入 |
| 职工信息自动获取 | 输入职工编号后自动获取姓名、部门、职业、技能等级等，不允许手动修改 | 考试人员新增、编辑 |
| 职工状态校验 | 添加考试人员时需校验职工状态是否为"在职"，离职或禁用状态不允许添加 | 考试人员新增、导入 |
| ruleType 统一规范 | 职业认定/竞赛/模拟考核统一使用 0/1/2 数值 | 组卷管理、组卷规则、资源题库 |

#### 4.1.1 模块1：账号管理

##### 4.1.1.1 模块概述
- **模块定位**：考核管理工具的基础模块，负责管理员和职工的全生命周期管理。
- **核心职责**：管理员列表管理、职工列表管理、用户信息编辑、密码修改、用户状态管理。
- **依赖关系**：为所有其他模块提供用户认证和权限支持，依赖考核站管理获取考核站信息，依赖部门管理获取部门信息。
- **前端组件**：`views/account/index.vue`、`views/student/index.vue`；activeView：`teacherList`、`student`。
