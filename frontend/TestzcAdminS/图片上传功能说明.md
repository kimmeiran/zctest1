# 图片上传功能说明

## 功能概述

本项目已集成图片上传功能，支持在题目内容和答案中添加图片附件。

## 后端接口

### 通用文件上传接口
- **URL**: `/api/upload`
- **方法**: `POST`
- **参数**: `file` (MultipartFile)
- **返回**: 
  ```json
  {
    "code": 200,
    "message": "上传成功",
    "data": "服务器文件路径"
  }
  ```

### 图片上传接口（推荐）
- **URL**: `/api/uploadImage`
- **方法**: `POST`
- **参数**: `file` (MultipartFile，仅限图片)
- **返回**: 
  ```json
  {
    "code": 200,
    "message": "上传成功",
    "data": "/api/image/文件名"
  }
  ```

### 图片访问接口
- **URL**: `/api/image/{fileName}`
- **方法**: `GET`
- **功能**: 直接访问上传的图片文件
- **特点**: 支持缓存，自动设置正确的Content-Type

### 下载接口
- **URL**: `/api/download`
- **方法**: `GET`
- **参数**: `filePath` (服务器文件路径)
- **功能**: 根据文件路径下载文件

## 前端使用

### 1. 导入API
```javascript
import { uploadImage, getImageUrl, validateImageFile } from '@/api/imageUpload.js'
```

### 2. 上传图片
```javascript
const handleImageUpload = async (event) => {
    const file = event.target.files[0]
    if (file) {
        // 验证文件
        const validation = validateImageFile(file)
        if (!validation.valid) {
            alert(validation.message)
            return
        }
        
        // 上传图片
        const result = await uploadImage(file)
        if (result.success) {
            // 直接使用返回的URL，无需转换
            const imageUrl = result.data
            console.log('图片上传成功:', imageUrl)
        } else {
            alert(result.message)
        }
    }
}
```

### 3. 显示图片
```html
<img :src="imageUrl" alt="图片" style="max-width:100%;max-height:120px;" />
```

## 文件验证规则

### 支持格式
- JPG/JPEG
- PNG
- GIF
- BMP
- WebP

### 文件大小限制
- 最大文件大小: 5MB

### 验证逻辑
1. 检查文件类型是否为图片
2. 检查文件大小是否超过限制
3. 检查文件扩展名是否支持

## 配置说明

### 后端配置
在 `application-dev.properties` 中配置图片存储路径：
```properties
imagePath=/Users/acker/File/project/test1/zcexam/src/main/resources/images
```

### 前端配置
前端通过 `/api/upload` 调用后端接口，确保后端服务运行在正确的端口和路径上。

## 使用示例

### 题目图片上传
1. 在题目内容下方点击"上传图片"按钮
2. 选择图片文件
3. 系统自动验证并上传
4. 上传成功后显示图片预览
5. 可以点击"移除图片"删除已上传的图片

### 答案图片上传
1. 在答案输入框下方点击"上传答案图片"按钮
2. 选择图片文件
3. 系统自动验证并上传
4. 上传成功后显示图片预览
5. 可以点击"移除图片"删除已上传的图片

## 注意事项

1. **文件存储**: 上传成功后，图片会保存在后端配置的 `imagePath` 目录下
2. **URL访问**: 图片上传成功后返回的URL可以直接访问，格式为 `/api/image/文件名`
3. **域名拼接**: 可以拼接域名直接访问，如：`http://localhost:7092/api/image/文件名`
4. **认证配置**: 图片访问接口已添加到JWT排除路径，无需认证即可访问
5. **文件唯一性**: 系统会自动为文件名添加时间戳前缀，避免重复
6. **错误处理**: 上传失败时会显示相应的错误提示
7. **数据持久化**: 图片URL会保存在表单数据中，提交时需要一并保存到数据库

## 故障排除

### 常见问题
1. **上传失败**: 检查后端服务是否正常运行
2. **图片无法显示**: 检查文件路径是否正确，文件是否存在
3. **文件大小超限**: 确保图片文件小于5MB
4. **格式不支持**: 确保使用支持的图片格式
5. **401认证错误**: 确保图片访问接口已添加到JWT排除路径
6. **端口不匹配**: 确保前端使用正确的后端服务地址和端口

### 调试方法
1. 查看浏览器控制台的错误信息
2. 检查后端日志中的错误信息
3. 验证文件路径和权限设置
