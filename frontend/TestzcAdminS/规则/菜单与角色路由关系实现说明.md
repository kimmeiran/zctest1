# 菜单与角色路由关系实现说明

> **文档说明**：本文档详细说明中车考核工具中菜单和角色的路由关系是如何实现的。

## 核心实现方式

### 1. 实现原理

**重要说明**：本项目**不是使用 Vue Router 进行路由跳转**，而是使用**组件切换**的方式实现页面切换。

**实现机制**：
- 使用 `activeView` 状态变量控制显示哪个组件
- 通过 `v-if` 指令根据 `activeView` 的值条件渲染不同组件
- 菜单点击时调用 `setActiveView()` 函数切换组件

## 实现架构

### 1.1 状态管理

```javascript
// 在 Main.vue 的 setup() 中
const activeMenu = ref("");      // 当前激活的菜单（用于子菜单展开）
const activeView = ref("");      // 当前激活的视图（控制显示哪个组件）
```

### 1.2 角色判断

```javascript
// 从 sessionStorage 获取用户信息
const users = sessionStorage.getItem("user")
  ? JSON.parse(sessionStorage.getItem("user") || "{}")
  : {};

// 角色判断（使用 computed 属性）
const isAdmin = computed(() => users.role === 6);              // 总管理员
const isManager = computed(() => users.role === 5);           // 管理
const isPaperExpert = computed(() => users.role === 7 || users.role === 8);  // 组卷/审卷专家
const isQuestionExpert = computed(() => users.role === 2 || users.role === 3); // 命题/审题专家
const isReviewExpert = computed(() => users.role === 3);      // 审题专家
const isQuestionSetter = computed(() => users.role === 2);     // 命题专家
const isExamStationAdmin = computed(() => users.role === 4);   // 考核站管理员
const isGradingExpert = computed(() => users.role === 1);     // 阅卷专家
```

## 菜单显示控制

### 2.1 菜单权限控制（模板层）

菜单显示通过 `v-if` 指令根据角色计算属性控制：

```vue
<template>
  <ul class="menu">
    <!-- 角色7或8：组卷/审卷专家 -->
    <template v-if="isPaperExpert">
      <li class="menu-item" :class="{ active: activeView === '手动组卷' }">
        <a href="#" @click="setActiveView('手动组卷')">组卷管理</a>
      </li>
    </template>

    <!-- 角色2或3：命题/审题专家 -->
    <template v-else-if="isQuestionExpert">
      <li class="menu-item" :class="{ active: activeView === '资源题库' }">
        <a href="#" @click="setActiveView('资源题库')">资源题库</a>
      </li>
      <!-- 命题专家还显示组卷管理 -->
      <li v-if="!isReviewExpert && !isQuestionSetter" 
          class="menu-item" 
          :class="{ active: activeView === '手动组卷' }">
        <a href="#" @click="setActiveView('手动组卷')">组卷管理</a>
      </li>
    </template>

    <!-- 角色4：考核站管理员 -->
    <template v-else-if="isExamStationAdmin">
      <li class="menu-item" :class="{ active: activeView === 'student' }">
        <a href="#" @click="setActiveView('student')">职工列表</a>
      </li>
      <!-- 更多菜单项... -->
    </template>

    <!-- 角色1：阅卷专家 -->
    <template v-else-if="isGradingExpert">
      <li class="menu-item" :class="{ active: activeMenu === '考核成绩' && isSubmenuView }">
        <a href="#" @click="toggleSubmenu('考核成绩')">考核成绩</a>
        <ul class="submenu" v-show="activeMenu === '考核成绩'">
          <li class="submenu-item" :class="{ active: activeView === '查看成绩' }">
            <a href="#" @click="setActiveView('查看成绩')">成绩列表</a>
          </li>
        </ul>
      </li>
    </template>

    <!-- 其他角色（总管理员、管理）：完整菜单 -->
    <template v-else>
      <!-- 账号管理（有子菜单） -->
      <li class="menu-item" :class="{ active: activeMenu === '账号管理' && isSubmenuView }">
        <a href="#" @click="toggleSubmenu('账号管理')">账号管理</a>
        <ul class="submenu" v-show="activeMenu === '账号管理'">
          <li v-if="isAdmin || isManager" 
              class="submenu-item" 
              :class="{ active: activeView === 'teacherList' }">
            <a href="#" @click="setActiveView('teacherList')">管理员列表</a>
          </li>
          <li class="submenu-item" :class="{ active: activeView === 'student' }">
            <a href="#" @click="setActiveView('student')">职工列表</a>
          </li>
        </ul>
      </li>
      <!-- 更多菜单项... -->
    </template>
  </ul>
</template>
```

## 核心函数实现

### 3.0 视图访问权限校验

切换视图前会校验**当前角色是否有权访问目标视图**，与菜单显示保持一致，避免通过本地缓存或其它方式进入无权限页面。

- **校验时机**：`setActiveView(view)` 被调用时、以及从 localStorage 恢复 `activeView` 时。
- **实现**：维护「角色 → 可访问视图列表」映射（与 2.1 菜单权限一致）；管理(5)、总管理(6) 不限制；其他角色仅能进入其可见菜单对应的视图。
- **无权限时**：不切换视图，并提示「无权限访问该页面」；恢复状态时若保存的视图对当前角色不可见，则不恢复该视图，由默认视图逻辑处理。

### 3.1 setActiveView() - 切换视图

```javascript
// 设置当前激活的视图（切换前校验当前角色是否有权访问该视图）
const setActiveView = view => {
  if (!canAccessView(view)) {
    ElMessage.warning('无权限访问该页面');
    return;
  }
  activeView.value = view;  // 更新 activeView，触发组件切换

  // 特殊处理：如果切换到"考试管理"，清除相关标记
  if (view === "考试管理") {
    sessionStorage.removeItem('showApplicantsFromExam');
    sessionStorage.removeItem('applicantsParams');
    sessionStorage.removeItem('fromExamManagement');
    directTeachExamId.value = null;
    showApplicantsFromExam.value = false;
  }

  // 根据视图设置对应的菜单（用于子菜单展开）
  if (view === "student" || view === "teacherList") {
    activeMenu.value = "账号管理";
  } else if (view === "查看成绩" || view === "成绩分析") {
    activeMenu.value = "考核成绩";
  } else if (view === "自动组卷" || view === "手动组卷" || view === "历史手动组卷") {
    activeMenu.value = "组卷管理";
  } else if (view === "资源题库") {
    activeMenu.value = "资源题库";
  } else {
    activeMenu.value = "";
  }
  
  saveState(); // 保存状态到 localStorage
};
```

### 3.2 toggleSubmenu() - 切换子菜单

```javascript
// 切换子菜单显示状态
const toggleSubmenu = menu => {
  // 如果点击的是当前已激活的菜单，则关闭它
  if (activeMenu.value === menu) {
    activeMenu.value = "";
    saveState();
    return;
  }

  // 否则，打开点击的菜单
  activeMenu.value = menu;

  // 根据选择的菜单设置默认视图
  switch (menu) {
    case "账号管理":
      if (isAdmin.value || isManager.value) {
        setActiveView("teacherList");
      } else {
        setActiveView("student");
      }
      break;
    case "考核成绩":
      setActiveView("查看成绩");
      break;
    case "组卷管理":
      setActiveView("自动组卷");
      break;
    default:
      break;
  }
  
  saveState();
};
```

### 3.3 状态持久化

```javascript
// 从 localStorage 恢复状态
const restoreState = () => {
  const savedMenu = localStorage.getItem("activeMenu");
  const savedView = localStorage.getItem("activeView");
  
  if (savedMenu) {
    activeMenu.value = savedMenu;
  }
  if (savedView) {
    activeView.value = savedView;
  }
};

// 保存状态到 localStorage
const saveState = () => {
  localStorage.setItem("activeMenu", activeMenu.value);
  localStorage.setItem("activeView", activeView.value);
};
```

## 初始化逻辑

### 4.1 默认视图设置

```javascript
// 根据角色设置默认视图
const defaultView = ref(
  isAdmin.value ? "teacherList" :                    // 总管理员：管理员列表
  isPaperExpert.value ? "手动组卷" :                 // 组卷/审卷专家：手动组卷
  isQuestionExpert.value ? "资源题库" :              // 命题/审题专家：资源题库
  isExamStationAdmin.value ? "student" :            // 考核站管理员：职工列表
  isGradingExpert.value ? "查看成绩" :              // 阅卷专家：查看成绩
  "student"                                          // 默认：职工列表
);
```

### 4.2 组件挂载时的初始化

```javascript
onMounted(() => {
  // 检查用户是否已登录
  if (!checkUserAuthentication()) {
    return;
  }

  // 尝试恢复之前保存的状态
  restoreState();
  
  // 如果没有保存的状态，则使用默认设置
  if (!activeView.value) {
    if (isQuestionExpert.value) {
      setActiveView(defaultView.value);
    } else if (isExamStationAdmin.value) {
      setActiveView(defaultView.value);
    } else if (isGradingExpert.value) {
      toggleSubmenu("考核成绩");
      setActiveView(defaultView.value);
    } else {
      toggleSubmenu("账号管理");
      setActiveView(defaultView.value);
    }
  }
});
```

## 数据流向图

```
用户登录
    ↓
从 sessionStorage 获取用户信息
    ↓
根据 users.role 计算角色判断属性
    ↓
根据角色显示对应菜单（v-if）
    ↓
用户点击菜单
    ↓
调用 setActiveView(view) 或 toggleSubmenu(menu)
    ↓
更新 activeView 和 activeMenu 状态
    ↓
根据 activeView 条件渲染对应组件（v-if）
    ↓
保存状态到 localStorage（持久化）
```

## 关键特点

### 1. 不是真正的路由跳转

- ❌ **不使用** `router.push()` 进行路由跳转
- ✅ **使用** `activeView` 状态变量控制组件显示
- ✅ **使用** `v-if` 指令条件渲染组件

### 2. 单页应用（SPA）

- 所有组件都在同一个页面中
- 通过组件切换实现"页面"切换
- URL 不会改变（除非使用真正的路由）

### 3. 状态持久化

- 使用 `localStorage` 保存菜单和视图状态
- 页面刷新后可以恢复之前的状态

### 4. 角色权限控制

- 在模板层通过 `v-if` 控制菜单显示
- 基于 `computed` 属性进行角色判断
- 不同角色看到不同的菜单项

## 与 Vue Router 的关系

### 实际使用的路由

项目中确实配置了 Vue Router（`src/router/index.ts`），但主要用于：

1. **登录页面跳转**：`/login`
2. **主页面跳转**：`/home` → `Main.vue`
3. **特殊页面**：
   - `/personal-center` - 个人中心
   - `/preview` - 文件预览
   - `/exam-distribute/:examId` - 考试派发

### 菜单切换不使用路由

- 菜单点击**不触发路由跳转**
- 菜单切换通过 `setActiveView()` 函数实现
- 组件切换通过 `v-if` 条件渲染实现

## 总结

### 实现方式

1. **状态驱动**：使用 `activeView` 和 `activeMenu` 状态变量
2. **条件渲染**：通过 `v-if` 指令根据状态显示不同组件
3. **角色控制**：通过 `v-if` 和 `computed` 属性控制菜单显示
4. **状态持久化**：使用 `localStorage` 保存和恢复状态

### 优点

- ✅ 实现简单，不需要复杂的路由配置
- ✅ 切换速度快，无需重新加载
- ✅ 状态可以持久化
- ✅ 权限控制清晰

### 缺点

- ❌ URL 不会改变，无法通过 URL 直接访问特定页面
- ❌ 浏览器前进/后退按钮无法使用
- ❌ 无法使用路由守卫进行权限控制
- ❌ 不利于 SEO（但后台系统不需要 SEO）
