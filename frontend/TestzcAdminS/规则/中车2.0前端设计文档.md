## 一、UI 组件与样式规范

### 1.1 组件库使用规范

**核心原则**：

✅ **可以使用 Element Plus 组件但需定制样式**，含义如下：

- **可以使用组件库的功能**：Element Plus 提供的表单、表格、弹窗、导航等功能都可以使用
- **必须定制视觉样式**：组件库的默认视觉样式不能直接使用，需要根据项目风格库文档进行定制
- **功能与样式分离**：使用组件库的功能特性（如数据绑定、事件处理、API方法等），但视觉样式必须符合项目风格库

**简单理解**：
- ✅ **允许**：使用 `<el-button>`、`<el-table>`、`<el-form>` 等组件及其功能
- ✅ **允许**：通过 CSS 覆盖组件的默认样式，使其符合项目风格库
- ❌ **禁止**：直接使用组件库的默认视觉样式（如默认颜色、默认圆角、默认间距等）

### 1.2 UI 组件库样式定制规范

#### 1.2.1 样式定制原则

| 定制方式 | 使用场景 | 优先级 | 说明 |
|---------|---------|--------|------|
| **CSS 变量覆盖** | 全局主题定制 | 高 | 通过 CSS 变量统一定制主题色、字体等 |
| **深度选择器** | 组件局部样式定制 | 中 | 使用 `:deep()` 或 `::v-deep` 定制组件内部样式 |
| **自定义类名** | 特定组件样式定制 | 中 | 通过 `class` 或 `custom-class` 属性定制 |
| **样式覆盖** | 临时样式调整 | 低 | 仅在必要时使用，需说明原因 |

#### 1.2.2 样式定制方法

**方法一：CSS 变量覆盖（推荐）**：

```css
/* assets/styles/variables.css */
:root {
  /* 覆盖 Element Plus 默认变量 */
  --el-color-primary: #c70019; /* 中车红 - 主色调 */
  --el-color-success: #67c23a;
  --el-color-warning: #e6a23c;
  --el-color-danger: #ff4d4f;
  --el-color-info: #909399;
  
  /* 字体大小 */
  --el-font-size-base: 14px;
  --el-font-size-large: 16px;
  --el-font-size-small: 12px;
  
  /* 圆角 */
  --el-border-radius-base: 4px;
  --el-border-radius-small: 2px;
  --el-border-radius-large: 8px;
  
  /* 间距 */
  --el-spacing-base: 8px;
  --el-spacing-small: 4px;
  --el-spacing-large: 16px;
}
```

**方法二：深度选择器定制（组件级）**：

```vue
<template>
  <el-button class="custom-button">按钮</el-button>
</template>

<style scoped>
/* ✅ 正确：使用深度选择器定制组件内部样式 */
.custom-button :deep(.el-button__inner) {
  font-weight: 500;
  letter-spacing: 0.5px;
}

.custom-button :deep(.el-button) {
  border-radius: 4px;
  padding: 8px 15px;
  background-color: #c70019;
  border-color: #c70019;
}
</style>
```

**方法三：自定义类名（特定场景）**：

```vue
<template>
  <el-button 
    class="custom-button"
    :class="{ 'custom-button--primary': isPrimary }"
  >
    按钮
  </el-button>
</template>

<style scoped>
.custom-button {
  /* 基础样式 */
}

.custom-button--primary {
  /* 特定样式 */
}
</style>
```

#### 1.2.3 样式定制检查清单

- [ ] 是否使用 CSS 变量进行全局主题定制？
- [ ] 组件样式是否根据风格库文档定制？
- [ ] 是否避免直接覆盖组件库默认样式？
- [ ] 样式定制是否有明确说明？
- [ ] 样式是否与设计稿一致？

### 1.3 UI 组件库与自定义组件的平衡

**使用决策原则**：

| 场景 | 使用方案 | 说明 |
|------|---------|------|
| **通用按钮** | Element Plus Button | 功能完善，样式定制 |
| **通用表单** | Element Plus Form | 表单验证、布局完善 |
| **通用表格** | Element Plus Table | 排序、筛选、分页功能 |
| **通用图标** | Element Plus Icons | 图标库丰富 |
| **个性化按钮** | 自定义组件 | 特殊样式、特殊交互 |
| **复杂业务组件** | 自定义组件 | 业务逻辑复杂 |
| **页面区块** | 自定义组件 | 页面特定区块 |

**决策流程**：

```
需要组件
    ↓
是通用功能？
    ├─ 是 → 使用 Element Plus
    │        ├─ 样式可定制？
    │        │   ├─ 是 → 使用 Element Plus + 样式定制
    │        │   └─ 否 → 考虑自定义组件
    │        └─ 功能满足？
    │            ├─ 是 → 使用 Element Plus
    │            └─ 否 → 考虑自定义组件
    │
    └─ 否 → 自定义组件
```

### 1.4 组件样式规范

#### 1.4.1 按钮组件样式规范

**按钮状态定义**：

| 状态 | 视觉表现 | 说明 |
|------|---------|------|
| **默认状态** | 主色调背景（#c70019），白色文字 | 正常可点击状态 |
| **悬停状态** | 主色调加深，轻微阴影 | 鼠标悬停时的反馈 |
| **点击状态** | 主色调进一步加深 | 点击时的即时反馈 |
| **禁用状态** | 灰色背景，灰色文字，透明度降低 | 不可点击状态 |
| **加载状态** | 显示加载图标，禁用点击 | 处理中状态 |

**按钮尺寸规范**：

| 尺寸 | 高度 | 内边距 | 字体大小 | 使用场景 |
|------|------|--------|---------|---------|
| **large** | 40px | 16px 32px | 16px | 主要操作按钮 |
| **default** | 36px | 12px 24px | 14px | 常规按钮 |
| **small** | 32px | 8px 16px | 12px | 次要按钮 |

**按钮样式示例**：

```vue
<template>
  <el-button class="primary-button" type="primary" size="default">
    提交
  </el-button>
</template>

<style scoped>
.primary-button :deep(.el-button) {
  background-color: #c70019; /* 中车红 */
  border-color: #c70019;
  color: white;
  border-radius: 4px;
  padding: 8px 15px;
  font-size: 14px;
  font-weight: 500;
  transition: all 150ms ease-out;
}

.primary-button :deep(.el-button:hover) {
  background-color: #a50014; /* 悬停时加深 */
  border-color: #a50014;
}

.primary-button :deep(.el-button:active) {
  background-color: #8b0010; /* 点击时进一步加深 */
  transform: scale(0.98);
  transition: all 150ms ease-out;
}
</style>
```

#### 2.1.2 页面元素布局规范

**顶部导航栏**：
- 宽度：100%
- 高度：60px
- 背景色：`#2c3e50`（深蓝灰）
- 文字颜色：白色
- 包含：Logo（"中国中车考核管理工具"）、用户角色、考核站名称、个人中心链接、用户名
- 位置：固定定位（`position: fixed`）或相对定位
- 内边距：`0 20px`
- 布局：`display: flex`，`justify-content: space-between`

**侧边栏导航**（后台管理系统）：
- 宽度：180px（固定）
- 最小宽度：180px
- 背景色：`#f5f7fa`（浅灰）
- 边框：右侧 `1px solid #eee`
- 包含：功能模块导航菜单
- 位置：相对定位
- 菜单项：
  - 内边距：`15px 20px`
  - 文字颜色：`#333`
  - 激活状态：背景色 `#c70019`，文字白色
  - 悬停状态：背景色 `#e6f7ff`

**主内容区域**：
- 宽度：自适应（100% - 侧边栏宽度）
- 背景色：白色（`#fff`）
- 内边距：0（内容区域内部组件自行控制）
- 包含：面包屑导航、页面标题、操作按钮、搜索筛选、数据展示、分页器
- 布局：`display: flex`，`flex-direction: column`

**面包屑导航**：
- 位置：主内容区域顶部
- 内边距：`16px 24px`
- 背景色：`#f5f7fa`
- 边框：底部 `1px solid #eee`
- 字体大小：14px
- 文字颜色：`#666`

**搜索/筛选区域**：
- 位置：主内容区域顶部（面包屑下方）
- 布局：横向排列，支持多条件筛选
- 包含：搜索框、筛选下拉框、筛选按钮、重置按钮
- 间距：组件之间使用 `gap: 10px` 或 `margin-right: 15px`

**数据展示区域**：
- 位置：搜索/筛选区域下方
- 布局：根据页面类型展示（表格、卡片、图表等）
- 支持：加载态、空态、错误态展示
- 内边距：根据页面需求设置（通常 `20px` 或 `24px`）

**分页器**：
- 位置：数据展示区域下方
- 包含：页码、每页数量选择、总数显示
- 对齐：通常右对齐或居中

## 三、交互设计

### 3.1 交互行为规范

#### 3.1.1 按钮交互

**按钮点击交互**：

| 交互阶段 | 视觉反馈 | 说明 |
|---------|---------|------|
| **悬停** | 背景色变化、轻微阴影 | 鼠标悬停反馈 |
| **点击** | 按下效果（scale 0.98） | 点击即时反馈 |
| **加载** | 显示加载图标 | 处理中状态 |
| **完成** | 恢复原状或跳转 | 操作完成反馈 |

**按钮交互示例**：

```vue
<template>
  <el-button 
    class="interactive-button"
    :loading="isLoading"
    @click="handleClick"
  >
    提交
  </el-button>
</template>

<style scoped>
.interactive-button {
  /* 悬停效果 */
  transition: all 150ms ease-out;
}

.interactive-button:hover {
  background-color: #a50014; /* 悬停时加深 */
  box-shadow: 0 2px 8px rgba(199, 0, 25, 0.2);
}

.interactive-button:active {
  transform: scale(0.98);
  background-color: #8b0010;
}
</style>
```

#### 3.1.2 表单交互

**表单输入交互**：

| 交互阶段 | 视觉反馈 | 说明 |
|---------|---------|------|
| **聚焦** | 边框高亮（#c70019）、轻微阴影 | 输入框获得焦点 |
| **输入** | 实时验证反馈 | 输入时显示验证结果 |
| **失焦** | 边框恢复正常 | 输入框失去焦点 |
| **验证错误** | 红色边框（#ff4d4f）、错误提示 | 验证失败时的反馈 |
| **验证成功** | 绿色边框（可选） | 验证成功时的反馈 |

**表单提交交互**：

| 交互阶段 | 视觉反馈 | 说明 |
|---------|---------|------|
| **提交中** | 按钮显示加载状态 | 禁用按钮，显示加载图标 |
| **提交成功** | 成功提示消息 | 显示成功提示，跳转或关闭 |
| **提交失败** | 错误提示消息 | 显示错误提示，保持表单状态 |

#### 3.1.3 列表交互

**列表项交互**：

| 交互行为 | 视觉反馈 | 说明 |
|---------|---------|------|
| **悬停** | 背景色变化（#e6f7ff） | 鼠标悬停行高亮 |
| **点击** | 选中状态 | 点击行选中 |
| **双击** | 打开详情 | 双击行打开详情页 |

#### 3.1.4 弹窗交互

**弹窗打开/关闭交互**：

| 交互阶段 | 视觉反馈 | 说明 |
|---------|---------|------|
| **打开** | 淡入 + 缩放 | 弹窗打开 |
| **关闭** | 淡出 + 缩放 | 弹窗关闭 |
| **取消按钮** | 必须关闭弹窗 | 点击取消立即关闭 |
| **确定按钮** | 提交后关闭 | 提交成功后关闭 |

**弹窗交互规范**：

- ✅ **取消按钮**：点击后必须关闭弹窗
- ✅ **确定按钮**：提交成功后关闭弹窗
- ✅ **遮罩层点击**：根据业务需求决定是否可关闭
- ✅ **ESC 键**：默认支持 ESC 键关闭（可选禁用）

## 四、设计系统同步机制

### 5.1 设计交付物规范

**设计交付物清单**：

| 交付物 | 格式 | 用途 | 交付时间 |
|--------|------|------|---------|
| **设计稿** | Figma/Sketch | 视觉设计参考 | 开发前 |
| **设计规范** | 文档/设计系统 | 设计规范参考 | 开发前 |
| **切图资源** | PNG/SVG | 图片资源 | 开发中 |
| **交互说明** | 文档/原型 | 交互逻辑说明 | 开发前 |
| **标注文件** | Figma/蓝湖 | 尺寸、颜色、间距标注 | 开发中 |

**设计稿要求**：

- ✅ **完整性**：设计稿应包含所有页面和状态
- ✅ **标注清晰**：尺寸、颜色、间距等标注清晰
- ✅ **资源导出**：切图资源按规范导出
- ✅ **交互说明**：交互逻辑说明完整
- ✅ **响应式设计**：包含不同屏幕尺寸设计

### 5.2 设计评审机制

**设计评审流程**：

```
设计初稿
    ↓
内部评审（设计团队）
    ↓
设计优化
    ↓
跨团队评审（产品、设计、开发）
    ↓
设计确认
    ↓
设计交付
    ↓
开发实现
    ↓
设计验收
```

**评审检查清单**：

- [ ] **视觉设计**
  - [ ] 设计是否符合风格库规范？
  - [ ] 颜色、字体、间距是否一致？
  - [ ] 视觉层次是否清晰？

- [ ] **交互设计**
  - [ ] 交互流程是否合理？
  - [ ] 交互反馈是否明确？

- [ ] **技术可行性**
  - [ ] 设计是否技术可行？
  - [ ] 是否有技术难点？
  - [ ] 是否需要技术方案调整？

### 5.3 设计验收标准

**验收检查清单**：

- [ ] **视觉一致性**
  - [ ] 颜色是否与设计稿一致？
  - [ ] 字体是否与设计稿一致？
  - [ ] 间距是否与设计稿一致？
  - [ ] 圆角、阴影等细节是否一致？

- [ ] **交互一致性**
  - [ ] 交互流程是否与设计一致？
  - [ ] 交互反馈是否与设计一致？

- [ ] **响应式一致性**
  - [ ] 不同屏幕尺寸是否与设计一致？
  - [ ] 移动端适配是否与设计一致？
