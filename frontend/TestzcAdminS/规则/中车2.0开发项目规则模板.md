# 中车1.5开发项目规则

本文档总结了中车1.5项目的开发规则和规范，供项目开发参考。

## 一、技术栈

### 1.1 核心框架与工具
- **Vue 3.3.4** - 使用Composition API和`<script setup>`语法
- **TypeScript 5.0.2** - 部分文件使用TypeScript，部分使用JavaScript（混合使用）
- **Vite 4.4.5** - 构建工具和开发服务器
- **Vue Router 4.2.4** - 路由管理（使用Hash模式）
- **Element Plus 2.9.11** - UI组件库（中文语言包）
- **Axios 1.9.0** - HTTP请求库

### 1.2 开发工具
- **vue-tsc** - TypeScript类型检查
- **Babel** - 代码转换（使用@vue/cli-plugin-babel/preset）
- **ESLint** - 代码检查（配置在package.json中）

### 1.3 其他依赖
- `@element-plus/icons-vue` - Element Plus图标库
- `@vue-office/docx`、`@vue-office/excel`、`@vue-office/pdf` - Office文件预览
- `html2canvas`、`jspdf` - PDF导出
- `xlsx` - Excel文件处理
- `lucide-vue-next` - 图标库

## 三、代码规范

### 3.1 组件开发规范

#### 3.1.1 组件写法
- **必须使用** Composition API和`<script setup>`语法
- 支持TypeScript和JavaScript混合使用（`.vue`文件可自由选择）
- 组件样式使用`scoped`作用域

```vue
<template>
  <div class="component-name">
    <!-- 模板内容 -->
  </div>
</template>

<script setup lang="ts">
// TypeScript写法
import { ref, computed, onMounted } from 'vue'

defineProps<{
  title: string;
  theme?: 'blue' | 'gold';
}>()

const count = ref(0)
</script>

<style scoped>
/* 样式内容 */
</style>
```

#### 3.1.2 组件命名
- 组件文件名使用**PascalCase**（如：`PageHeader.vue`、`InputField.vue`）
- 组件内部class使用**kebab-case**（如：`page-header`、`input-field`）

### 3.2 API接口开发规范

#### 3.2.1 API文件结构
- API文件统一放在`src/api/`目录
- 可使用`.js`或`.ts`文件（混合使用）
- 每个业务模块一个文件（如：`account.js`、`user.ts`、`exam.js`）

#### 3.2.2 API请求配置

**基础配置**：
```javascript
// 使用Vite环境变量
// 开发环境
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || "/exam"
// 生产环境
// const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || "/exam/api"

// 确保axios使用相对路径
axios.defaults.baseURL = ''
```

**统一请求头**：
```javascript
headers: {
  "Content-Type": "application/json",
  Authorization: `${localStorage.getItem("token")}`,
}
```

#### 3.2.3 API响应格式
- 统一响应格式：`{ code: number, msg: string, data?: any }`
- `code === 200` 表示成功
- 分页数据格式：`{ rows: [], total: number }` 或 `{ list: [], total: number }`

#### 3.2.4 API函数规范

**函数定义**：
```javascript
/**
 * 函数功能描述
 * @param {类型} param - 参数说明
 * @returns {Promise<Response>} 响应数据
 */
export const functionName = async (params): Promise<Response> => {
  try {
    const response = await axios.post(`${API_BASE_URL}/api/endpoint`, params, {
      headers: {
        "Content-Type": "application/json",
        Authorization: `${localStorage.getItem("token")}`,
      },
    })
    return response.data
  } catch (error) {
    console.error("操作失败:", error)
    throw error
  }
}
```

**常见API操作模式**：
- GET请求：使用`axios.get`，参数通过`params`传递
- POST请求：使用`axios.post`，数据放在请求体
- PUT请求：使用`axios.put`
- DELETE请求：使用`axios.delete`，参数通过`params`或`data`传递
- 文件上传：使用`FormData`，设置`Content-Type: multipart/form-data`
- 文件下载：设置`responseType: 'blob'`

### 3.3 路由配置规范

#### 3.3.1 路由模式
- 使用**Hash模式**（`createWebHashHistory`）

#### 3.3.2 路由配置
```typescript
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    {
      path: '/path',
      name: 'RouteName',
      component: () => import('../views/Component.vue')
    }
  ]
})

export default router
```

### 3.4 TypeScript使用规范

#### 3.4.1 类型定义
- 接口类型定义使用`interface`
- 类型定义文件放在`src/types/`目录
- 可为第三方库补充类型定义（如：`element-plus.d.ts`）

#### 3.4.2 路径别名
- `@/*` → `src/*`
- `@question/*` → `src/views/question/*`

配置位置：
- `tsconfig.json` - TypeScript路径配置
- `jsconfig.json` - JavaScript路径配置
- `vite.config.ts` - Vite路径别名配置

#### 3.4.3 常见TypeScript错误处理

**错误一：未使用的导入**
- **错误信息**：`'XXX' is declared but its value is never read`
- **原因**：导入了类型或变量但未使用
- **解决方法**：
  - 移除未使用的导入
  - 如果确实需要保留，可以使用 `// eslint-disable-next-line @typescript-eslint/no-unused-vars` 注释
  ```typescript
  // 错误示例
  import { type Dictionary, type Department } from "@/api/user";
  // 如果 Dictionary 未使用，移除它
  import { type Department } from "@/api/user";
  ```

**错误二：对象属性不存在**
- **错误信息**：`Property 'xxx' does not exist on type '...'`
- **原因**：访问了类型定义中不存在的属性
- **解决方法**：
  - 使用类型断言 `(obj as any).propertyName`
  - 使用可选链 `obj?.propertyName`
  - 更新类型定义，添加缺失的属性
  ```typescript
  // 错误示例
  const userData = response.data.data;
  formData.titleLevel = userData.titleLevel; // 如果 userData 类型中没有 titleLevel
  
  // 解决方法1：使用类型断言
  formData.titleLevel = (userData as any).titleLevel || null;
  
  // 解决方法2：使用可选链（如果属性可能不存在）
  formData.titleLevel = userData?.titleLevel || null;
  
  // 解决方法3：更新接口定义（推荐）
  interface UserData {
    // ... 其他属性
    titleLevel?: string | null; // 添加可选属性
  }
  ```

**错误三：类型不匹配**
- **错误信息**：`Type 'X' is not assignable to type 'Y'`
- **原因**：赋值时类型不匹配
- **解决方法**：
  - 使用类型转换
  - 检查类型定义是否正确
  - 使用联合类型或泛型

### 3.5 状态管理

#### 3.5.1 用户信息存储
- Token存储在`localStorage`：`localStorage.getItem("token")`
- 用户信息存储在`sessionStorage`或`localStorage`
- 示例：`sessionStorage.getItem("user")`

#### 3.5.2 组件状态
- 使用`ref`和`reactive`管理组件内部状态
- 使用`computed`计算属性
- 使用`watch`监听数据变化

### 3.6 错误处理规范

#### 3.6.1 API错误处理
```javascript
try {
  const response = await apiFunction(params)
  const responseData = response.data || {}
  
  if (responseData.code === 200) {
    // 成功处理
  } else {
    throw new Error(responseData?.msg || "操作失败")
  }
} catch (error) {
  console.error("操作失败:", error)
  ElMessage.error(error.message || "操作失败，请稍后重试")
}
```

#### 3.6.2 错误提示
- 使用Element Plus的`ElMessage`组件进行消息提示
- 成功：`ElMessage.success("操作成功")`
- 警告：`ElMessage.warning("警告信息")`
- 错误：`ElMessage.error("错误信息")`

### 3.7 表单验证规范

#### 3.7.1 验证工具
- 验证函数统一放在`src/utils/validators.js`
- 提供常用的验证方法：
  - `validateRequired` - 必填验证
  - `validateNumber` - 数字验证
  - `validateName` - 姓名验证
  - `validateEmail` - 邮箱验证
  - `validatePhone` - 手机号验证
  - `validateLength` - 长度验证
  - `validateMatch` - 匹配验证
  - `validateForm` - 表单整体验证

#### 3.7.2 验证使用示例
```javascript
import { validateRequired, validateNumber } from '@/utils/validators'

const validateAccount = () => {
  const nameError = validateRequired(form.value.name, '账号')
  if (nameError) {
    ElMessage.warning(nameError)
    return false
  }
  return true
}
```

### 3.8 样式规范

#### 3.8.1 CSS作用域
- 组件样式使用`<style scoped>`
- 全局样式定义在`App.vue`中

#### 3.8.2 样式命名
- 使用**BEM命名法**或**kebab-case**
- class名称使用小写字母和连字符（如：`page-header`、`search-area`）

#### 3.8.3 响应式设计
- 使用媒体查询实现响应式布局
- 常用断点：
  - `@media (max-width: 768px)` - 平板
  - `@media (max-width: 480px)` - 手机

### 3.9 文件上传/下载规范

#### 3.9.1 文件上传
```javascript
const formData = new FormData()
formData.append('file', file)

const response = await axios.post(`${API_BASE_URL}/api/upload`, formData, {
  headers: {
    'Content-Type': 'multipart/form-data',
    Authorization: `${localStorage.getItem('token')}`,
  },
})
```

#### 3.9.2 文件下载
```javascript
const response = await axios.get(`${API_BASE_URL}/api/download`, {
  params: { filePath },
  responseType: 'blob',
  headers: {
    Authorization: `${localStorage.getItem('token')}`,
  },
})

// 创建下载链接
const url = window.URL.createObjectURL(new Blob([response.data]))
const link = document.createElement('a')
link.href = url
link.setAttribute('download', filename)
document.body.appendChild(link)
link.click()
```

## 五、开发约定

### 5.1 代码风格
- **缩进**：使用2个空格
- **引号**：JavaScript/TypeScript使用双引号，HTML属性使用双引号
- **分号**：JavaScript代码不使用分号（项目实践）
- **注释**：使用JSDoc格式编写函数注释

### 5.2 命名约定
- **变量/函数**：camelCase（如：`userName`、`fetchData`）
- **组件名**：PascalCase（如：`UserList`、`PageHeader`）
- **文件名**：组件使用PascalCase，其他使用camelCase或kebab-case
- **常量**：UPPER_SNAKE_CASE（如：`API_BASE_URL`）

### 5.3 文件组织
- **页面视图**：按功能模块组织在`src/views/`目录
- **组件**：可复用组件放在`src/components/`
- **API**：按业务模块分文件，统一放在`src/api/`
- **工具函数**：放在`src/utils/`
- **类型定义**：放在`src/types/`

### 5.4 组件通信
- **Props**：父组件向子组件传递数据
- **Emit**：子组件向父组件发送事件
- **Provide/Inject**：跨层级组件通信（如有需要）
- **EventBus**：未使用（Vue 3推荐使用其他方案）

### 5.5 分页处理

**分页参数格式**：
```javascript
{
  offset: 0,    // 偏移量（(currentPage - 1) * pageSize）
  limit: 10,    // 每页数量
  // 其他查询条件
}
```

**分页响应格式**：
```javascript
{
  code: 200,
  msg: "success",
  data: {
    rows: [],      // 数据列表（或 list）
    total: 100     // 总数量
  }
}
```

### 5.6 权限控制

**基于角色的权限控制**：
- 通过`sessionStorage`或`localStorage`存储用户角色
- 在组件中使用`v-if`控制功能显示
- 角色类型（示例）：
  - 1: 阅卷专家
  - 2: 命题专家
  - 3: 审题专家
  - 4: 考核站管理员
  - 5: 管理(可以查看考试题目)
  - 6: 总管理员(所有权限)

## 七、常见问题处理

### 7.1 跨域问题
- 开发环境：通过Vite proxy解决
- 生产环境：通过服务器代理解决

### 7.2 Token认证
- Token存储在`localStorage`
- 每次请求在Header中携带：`Authorization: ${localStorage.getItem("token")}`
- 注意：Authorization值前可能有空格，需根据实际情况调整

### 7.3 文件预览
- Word/Excel/PDF文件使用`@vue-office/*`系列组件预览

### 7.4 导出功能
- PDF导出：使用`html2canvas`和`jspdf`
- Excel导出：使用`xlsx`库
- Word导出：调用后端接口，返回blob数据

### 7.5 TypeScript编译错误处理

**常见编译错误及解决方案**：

1. **未使用的导入**
   - 问题：导入的类型或变量未使用
   - 解决：移除未使用的导入，保持代码整洁

2. **对象属性不存在**
   - 问题：访问了类型定义中不存在的属性
   - 解决：使用类型断言 `(obj as any).property` 或更新类型定义

3. **类型不匹配**
   - 问题：赋值时类型不匹配
   - 解决：使用类型转换或更新类型定义

**构建时检查**：
- 运行 `npm run build` 时会执行 `vue-tsc` 进行类型检查
- 确保所有TypeScript错误都已修复后再提交代码
- 如果确实需要临时忽略某些类型错误，可以使用 `// @ts-ignore` 注释（不推荐）
