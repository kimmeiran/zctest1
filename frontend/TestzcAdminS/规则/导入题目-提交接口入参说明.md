# 导入题目 - 提交接口入参说明

## 接口信息

**接口地址**：`POST /exam/api/questionBank/create`

**接口函数**：`createQuestionBank`（位于 `src/api/questionBank.js`）

**调用位置**：`src/views/question/ImportQuestion.vue` → `submitQuestionBankData` → `createQuestionBank`

---

## 一、主要入参结构

```javascript
{
  questionName: string,        // 题库名称（必填）
  questionType: string,        // 职业(工种)（必填）
  level: string,              // 技能等级（必填，对应表单的 difficulty）
  domain: number,             // 领域（必填）
  questions: Array,            // 题目列表（必填）
  baseType: number,           // 题库类型：0=职业认定，1=竞赛，2=模拟考核（必填）
  questionBankType: number,   // 题库类型（与 baseType 相同，必填）
  stationId?: string          // 考核站ID（可选，仅考核站管理员创建时传入）
}
```

---

## 二、参数详细说明

### 2.1 基础信息参数

| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `questionName` | string | ✅ | 题库名称，用户输入 | `"钳工高级工题库"` |
| `questionType` | string | ✅ | 职业(工种)，从字典表获取（type: 0） | `"钳工"` |
| `level` | string | ✅ | 技能等级，从字典表获取（type: 1），对应表单的 `difficulty` 字段 | `"高级工"` |
| `domain` | number | ✅ | 领域：<br>0=全部<br>2=职业道德<br>（其他领域值：1=基础知识，3=专业知识，4=相关知识，5=专业技能，6=相关技能） | `2` |
| `baseType` | number | ✅ | 题库类型：<br>0=职业认定<br>1=竞赛<br>2=模拟考核 | `0` |
| `questionBankType` | number | ✅ | 题库类型（与 `baseType` 相同） | `0` |
| `stationId` | string | ❌ | 考核站ID，仅当当前用户为考核站管理员时传入（从 `sessionStorage.user.stationId` 获取） | `"station123"` |

---

### 2.2 题目列表参数（questions）

`questions` 是一个数组，每个元素代表一道题目，结构如下：

```javascript
{
  content: string,                    // 题目内容（必填）
  type: string,                      // 题型名称（必填）
  answer: string,                    // 答案（必填）
  options: Array<string>,            // 选项数组（选择题必填，其他题型为空数组）
  domain: number,                    // 领域（必填，默认值：1）
  easeOrDifficulty: string,          // 难易程度（必填，默认值：'2'）
  importance: string,                // 重要程度（必填，默认值：'X'）
  number: number,                    // 题号（必填，默认值：0）
  
  // 以下为可选字段（有图片时包含）- 使用下划线命名
  answer_image?: Array<ImageObject>,           // 答案图片数组（可选）
  option_images?: Array<ImageObject>,           // 选项图片数组（可选）
  attached_image?: Array<ImageObject>,          // 题目附件图片数组（可选）
  question_content_images?: Array<ImageObject>   // 题目内容图片数组（可选）
}
```

---

### 2.3 题目对象字段说明

#### 2.3.1 基础字段

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `content` | string | ✅ | 题目内容文本，可能包含 `[img]` 占位符 | `"已知轴径为[img]，孔径为[img]..."` |
| `type` | string | ✅ | 题型名称，必须与后端定义的题型一致 | `"单项选择题"`、`"填空题"`、`"综合题"` 等 |
| `answer` | string | ✅ | 答案内容：<br>- 单选题：单个字母（如 `"A"`）<br>- 多选题：逗号分隔的字母（如 `"A,B,C"`）<br>- 填空题：逗号分隔的答案（如 `"答案1,答案2"`）<br>- 其他题型：答案文本，可能包含 `[img]` 占位符 | `"A"`、`"A,B"`、`"这是答案[img]"` |
| `options` | Array<string> | ✅ | 选项数组（选择题必填，其他题型为空数组）：<br>- 单选题/多选题：`["选项A", "选项B", "选项C", "选项D"]`<br>- 其他题型：`[]` | `["选项A", "选项B"]` 或 `[]` |
| `domain` | number | ✅ | 领域编号（默认值：1） | `1`、`2`、`3` 等 |
| `easeOrDifficulty` | string | ✅ | 难易程度（默认值：'2'）：<br>1=易<br>2=较易<br>3=中等<br>4=较难<br>5=难 | `"2"` |
| `importance` | string | ✅ | 重要程度（默认值：'X'）：<br>X=核心要素<br>Y=一般要素<br>Z=辅助要素 | `"X"` |
| `number` | number | ✅ | 题号，从 Word 文档中解析得到（默认值：0） | `1`、`2`、`3` 等 |

#### 2.3.2 图片字段（可选）

所有图片字段都是数组格式，每个元素为 `ImageObject` 对象：

```javascript
{
  url: string,      // 图片路径（必填）
  width?: number,   // 图片宽度，像素（可选）
  height?: number   // 图片高度，像素（可选）
}
```

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| `answer_image` | Array<ImageObject> | ❌ | 答案中的图片数组，用于替换答案文本中的 `[img]` 占位符，按顺序对应 | `[{url: "/images/answer1.png", width: 100, height: 50}]` |
| `option_images` | Array<ImageObject> | ❌ | 选项图片数组，按选项顺序对应（A/B/C/D...），仅选择题有 | `[{url: "/images/optionA.png"}, {url: "/images/optionB.png"}]` |
| `attached_image` | Array<ImageObject> | ❌ | 题目附件图片数组，显示在题目内容下方（如"补画下图的三视图"中的图） | `[{url: "/images/attached1.png", width: 200, height: 150}]` |
| `question_content_images` | Array<ImageObject> | ❌ | 题目内容图片数组，用于替换题目内容中的 `[img]` 占位符，按顺序对应 | `[{url: "/images/content1.png", width: 80, height: 21}, {url: "/images/content2.png"}]` |

**图片字段说明**：
- **字段命名**：所有图片字段使用下划线命名（`answer_image`、`option_images`、`attached_image`、`question_content_images`）
- 如果题目/答案中没有图片，则不包含对应的图片字段
- 图片数组中的顺序很重要：
  - `question_content_images`：按题目内容中 `[img]` 出现的顺序
  - `answer_image`：按答案中 `[img]` 出现的顺序
  - `option_images`：按选项顺序（A/B/C/D...）
  - `attached_image`：按文档中出现的顺序

## 四、数据流转过程

### 4.1 用户操作流程

```
1. 用户填写表单信息（questionName, questionType, difficulty, questionBankType, domain）
   ↓
2. 用户上传题目 Word 文件 → 调用 importQuestionBank(file, 1)
   ↓
3. 后端解析 Word 文件，返回解析结果（questionResult）
   ↓
4. 用户上传答案 Word 文件 → 调用 importQuestionBank(file, 2)
   ↓
5. 后端解析 Word 文件，返回解析结果（answerResult）
   ↓
6. 用户点击"提交"按钮 → 调用 handleSubmit()
   ↓
7. extractQuestionsFromImportResult() 合并题目和答案数据
   ↓
8. submitQuestionBankData() 构建请求参数
   ↓
9. createQuestionBank() 调用后端接口创建题库
```

### 4.2 数据合并逻辑（extractQuestionsFromImportResult）

**函数位置**：`src/views/question/ImportQuestion.vue` → `extractQuestionsFromImportResult`

**处理逻辑**：
1. 遍历题目数据（`questionResult.data.parsedContent.questionTypes`）
2. 为每道题目创建基础对象（content, type, answer, options, domain, easeOrDifficulty, importance, number）
3. 从答案数据中查找对应题号的答案，合并到题目对象中
4. 处理图片字段：
   - `answerImage`：从答案数据中提取
   - `optionImages`：从题目数据中提取
   - `attachedImage`：从题目数据中提取
   - `questionContentImages`：从题目数据中提取
5. 返回完整的题目数组

## 六、注意事项

### 6.1 必填字段校验

提交前会校验以下字段：
- ✅ `questionName`：题库名称
- ✅ `questionType`：职业(工种)
- ✅ `difficulty`：技能等级
- ✅ `questionBankType`：题库类型（不能为 null/undefined/空字符串）
- ✅ `domain`：领域（不能为 null/undefined/空字符串）

### 6.2 图片字段处理

1. **图片字段为可选**：如果题目/答案中没有图片，不包含对应字段
2. **图片数组顺序**：必须与 `[img]` 占位符出现的顺序一致
3. **图片对象格式**：统一为 `{url, width?, height?}` 格式
4. **兼容性处理**：支持 `optionImages` / `option_images`、`attachedImage` / `attached_image` 等不同命名

### 6.3 默认值处理

如果导入结果中某些字段缺失，会使用以下默认值：
- `domain`：`1`（基础知识）
- `easeOrDifficulty`：`'2'`（较易）
- `importance`：`'X'`（核心要素）
- `number`：`0`
- `answer`：`''`（空字符串）
- `options`：`[]`（空数组）

### 6.4 考核站管理员特殊处理

如果当前用户是考核站管理员（`sessionStorage.user.stationId` 存在），会自动在 `extra` 对象中添加 `stationId` 字段。
