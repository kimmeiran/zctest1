# 中车考核工具菜单项梳理

> **文档说明**：本文档梳理了系统中所有菜单项及其对应的角色权限、视图组件和功能说明。

## 角色说明

| 角色ID | 角色名称 | 说明 |
|--------|---------|------|
| 1 | 阅卷专家 | 负责阅卷工作 |
| 2 | 命题专家 | 负责题目命题 |
| 3 | 审题专家 | 负责题目审核 |
| 4 | 考核站管理员 | 考核站管理 |
| 5 | 管理 | 可以查看考试题目 |

| 7 | 组卷专家 | 负责组卷工作 |
| 8 | 审卷专家 | 负责审卷工作 |

## 菜单显示逻辑说明

### 1. 角色判断逻辑

```javascript
// 角色判断
isAdmin = role === 6        // 总管理员
isManager = role === 5      // 管理
isPaperExpert = role === 7 || role === 8  // 组卷/审卷专家
isQuestionExpert = role === 2 || role === 3  // 命题/审题专家
isReviewExpert = role === 3  // 审题专家
isQuestionSetter = role === 2  // 命题专家
isExamStationAdmin = role === 4  // 考核站管理员
isGradingExpert = role === 1  // 阅卷专家
```

### 2. 菜单显示优先级

菜单显示按以下顺序判断：

1. **组卷/审卷专家**（role=7或8）→ 仅显示"组卷管理"
2. **命题/审题专家**（role=2或3）→ 显示"资源题库"，命题专家还显示"组卷管理"
3. **考核站管理员**（role=4）→ 显示考核站相关功能菜单
4. **阅卷专家**（role=1）→ 仅显示"考核成绩"
5. **其他角色**（总管理员、管理）→ 显示完整菜单

### 3. 子菜单显示逻辑

- **账号管理**：总管理员和管理显示完整子菜单（管理员列表+职工列表），其他角色仅显示职工列表
- **考核成绩**：所有角色都显示"成绩列表"子菜单

---

## 注意事项

1. **菜单项命名**：菜单项使用中文名称，视图标识使用英文或中文标识
2. **组件路径**：所有组件都在 `src/views/` 目录下
3. **权限控制**：菜单显示通过 `v-if` 指令控制，基于用户角色
4. **状态保存**：菜单激活状态保存在 `localStorage` 中
5. **默认视图**：不同角色登录后显示不同的默认视图
