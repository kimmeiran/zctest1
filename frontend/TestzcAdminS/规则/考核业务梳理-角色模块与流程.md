# 考核业务梳理：角色、模块与流程

> **梳理依据**：根据「考核站申请→组卷→审卷→派发→阅卷→成绩→证书」业务描述整理，并与《需求1-29前端调整说明》《中车2.0需求实现清单》对应。

---

## 一、角色与职责

| 角色 | 职责摘要 | 涉及业务点（对应你提供的条号） |
|------|----------|--------------------------------|
| **考核站管理员** | 线下收集考试信息，发起试卷申请；查看/下载派发试卷；确认证书；竞赛时上传竞赛题库、自行组卷并提交审核；线上考试时添加考试人员并派发试卷 | 1、8、12、13、14、15 |
| **管理员（总管理员/管理）** | 查看试卷申请；组卷；提交审卷；审卷通过后选择考核站批量派发；查看派发记录；证书信息新增与管理；技能考核试卷上传；制定组卷规则 | 2、3、4、6、7、10、11、16、17 |
| **审卷专家** | 对管理员提交的试卷进行审核 | 5 |
| **阅卷专家** | 线上考试：判分；线下考试：成绩导入；判分完成后状态为「已判」并有颜色区分 | 9 |

## 三、主流程与分支

### 3.1 主流程（职业认定 / 常规考试）

```
考核站管理员线下收集考试信息
    ↓
【1】试卷申请模块：新增申请，上传文件 → 向管理员发起试卷申请
    ↓
【2】管理员在考试管理-派发页面 ① 区域：查看试卷申请
    ↓
【3】管理员在组卷管理 ①：根据申请需要，自动组卷或手动组卷
    ↓
【4】组卷完成后，在组卷管理 ②：点击提交审卷
    ↓
【5】审卷专家审核
    ↓
【6】审卷通过后，管理员在派发页面 ③：选择考核站，批量派发
    ↓
【7】管理员在派发页面 ② 区域：查看派发记录
    ↓
【8】考核站管理员在派发页面：查看、下载接收到的试卷
    ↓
【9】阅卷：线上考试 → 阅卷专家判分；线下考试 → 阅卷专家成绩导入；判分完成显示「已判」+ 颜色
    ↓
【10】管理员、考核站管理员在考核成绩模块：查看成绩
    ↓
【11】成绩通过 → 证书线下邮寄；管理员在证书管理：新增证书信息、管理
    ↓
【12】考核站管理员在证书管理：确认证书状态（是否已收到）
```

### 3.2 分支一：竞赛类型

| 环节 | 说明 |
|------|------|
| 题库 | 【13】竞赛类型考试时，考核站管理员在**资源题库**上传**竞赛类型**题库 |
| 组卷 | 【14】竞赛类型时，考核站管理员可在**组卷管理**自行组卷，并可**提交审核**（走审卷流程） |

### 3.3 分支二：线上考试

| 环节 | 说明 |
|------|------|
| 考试人员与派发 | 【15】线上考试时，考核站管理员在**考试管理**对相应考试场次**添加考试人员**并**派发试卷** |

### 3.4 其他独立能力

| 条号 | 说明 |
|------|------|
| 16 | 管理员在**技能考核**模块上传技能考核试卷 |
| 17 | 管理员在**组卷规则**模块制定组卷规则 |

## 五、梳理结论（能梳理出来的内容）

1. **角色**：考核站管理员、管理员（总管理员/管理）、审卷专家、阅卷专家；各自在申请、组卷、审卷、派发、阅卷、成绩、证书上的职责清晰。
2. **模块**：试卷申请、考试管理（含派发① ② ③）、组卷管理（① ②）、审卷、考核成绩、证书管理、资源题库、组卷规则、技能考核；以及「考试管理-考试场次」下的添加人员与派发（线上考试）。
3. **主流程**：申请 → 管理员看申请 → 组卷 → 提交审卷 → 审卷 → 批量派发 → 派发记录 / 考核站下载 → 阅卷（线上判分/线下导入）→ 成绩查看 → 证书新增与管理 → 考核站确认证书。
4. **分支**：竞赛（题库上传、考核站自组卷+提交审核）；线上考试（考核站添加考试人员并派发试卷）。
5. **与需求/实现清单的对应**：上述 17 条均可对应到需求1–29及实现清单中的页面（资源题库、组卷规则、组卷管理、考试管理-派发、考试管理-报考人员、证书管理、技能考核、考核成绩等），便于按「逐页开发确认」落地。

如需，可在此基础上再拆成「按角色可进菜单」或「按页面验收要点」两份清单，便于前端与产品对焦。
