> **依据规范**：本清单严格按《中车2.0前端开发规则》**第九章「开发流程规范」**执行，采用 **9.1 逐页开发确认流程**。

## 与开发规则的对应关系

| 开发规则（9.1 逐页开发确认流程） | 本清单执行方式 |
|----------------------------------|----------------|
| 1. 页面规划阶段：确定页面列表和优先级 | 下方「实现顺序总览」即页面列表与优先级 |
| 2. 单页开发阶段：创建→布局→组件→样式→交互 | 各页面「具体改动」按此顺序实现，代码规范遵循《中车2.0前端开发规则》第一～八章 |
| 3. 页面自检阶段：功能 / 样式 / 代码 / 兼容性检查 | 每页完成后执行「自检清单」，通过后再进入确认 |
| 4. 页面确认阶段：提交确认→问题修复→确认通过 | 每页列出「验收要点」，确认通过后将该页状态改为「✅ 已完成」 |
| 5. 下一个页面开发 | **仅在当前页面确认通过后**才开发下一页 |

**其他规范**：
- 技术栈、目录结构、API 放置、命名等：见《中车2.0前端开发规则》第一～六章
- UI/样式：见《中车2.0前端设计文档》
- 需求字段与接口细节：可对照《中车2.0开发需求模板》填写；本清单实现细节见《需求1-29前端调整说明》
- **流程与原型**：流程步骤、原型对应、开发阶段详见《流程与原型对应及开发规划》

## 页面1：资源题库

**对应需求**：1、2、3、4、5

**涉及文件**：
- `src/views/question/QuestionModal.vue` — 新增/编辑弹窗增加「题库类型」
- `src/views/question/QuestionList.vue` — 列表增加「题库类型」列，去掉「审核状态」列
- `src/views/question/QuestionTable.vue` — 查看题目页显示「题库类型」
- `src/views/BasicExamManage.vue` — 保存时校验并传递 `questionBankType`
- `src/api/user.ts` — 导入接口增加 `questionBankType` 参数（若后端已支持）

**具体改动**：

1. **QuestionModal.vue**
   - 在「技能等级」后增加表单项：题库类型（必填），选项：竞赛 / 模拟考试 / 职业认定
   - 新增/重置时 `localExam.questionBankType` 置空
   - 保存时把 `questionBankType` 一并传给父组件

2. **QuestionList.vue**
   - 表头：在「管理人」后增加「题库类型」，删除「审核状态」
   - 表格行：增加 `exam.questionBankType` 列，删除审核状态列
   - 操作列：保留「查看题目」；提交审核/通过/拒绝/删除按需求可保留（权限改造后由「管理」操作），本页仅做列展示调整
   - 无数据时 `colspan` 改为 8（因少一列）

3. **QuestionTable.vue**
   - 在题目管理头部（如 `header-sub` 或题库名称下方）增加展示：题库类型：`examInfo.questionBankType || '-'`

4. **BasicExamManage.vue**
   - `handleModalSave`：校验 `exam.questionBankType` 必填
   - `currentExam` / 传给 QuestionModal 的 exam 中增加 `questionBankType`
   - `importQuestionsToBase` 调用时传入 `questionBankType`，并在调用 `importQuestions` 时传给后端（见 api）

5. **api/user.ts**
   - `importQuestions` 增加参数 `questionBankType?: string`，在 FormData 中 append `questionBankType`

**验收要点**：导入时能选择题库类型并保存；列表显示题库类型且无审核状态列；进入查看题目能看到题库类型。

**自检清单**（开发完成后必须执行，与开发规则 9.1 一致）：
- [ ] 功能检查：导入/列表/查看题目功能完整，交互正常
- [ ] 样式检查：符合项目风格，无错位
- [ ] 代码检查：`npm run build` 通过，无 ESLint 报错
- [ ] 兼容性检查：目标浏览器可正常使用

**确认**：自检通过后提交确认；获得确认后，将本页状态改为「✅ 已完成」，再进入页面2。
