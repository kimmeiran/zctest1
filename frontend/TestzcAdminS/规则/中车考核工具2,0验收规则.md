## 一、验收流程

**验收阶段**：
1. **开发自检**：开发完成后，开发人员按照验收标准进行自检
2. **内部验收**：项目负责人或技术负责人进行内部验收
3. **产品验收**：产品经理或业务负责人进行功能验收
4. **最终验收**：所有验收通过后，页面标记为"验收通过"

**验收方式**：
- **视觉验收**：对照设计文档和项目规范，检查视觉效果
- **功能验收**：测试所有功能点，验证交互逻辑
- **性能验收**：使用工具测试性能指标
- **兼容性验收**：在不同浏览器和设备上测试
- **代码验收**：检查代码质量和规范

## 二、视觉验收标准（权重：25%）

**验收内容**：

| 验收项 | 验收标准 | 评分规则 | 满分 |
|-------|---------|---------|------|
| **色彩使用** | 主色调（中车红 #c70019）、辅助色、中性色使用符合设计文档 | 每发现1处不符合扣5分，扣完为止 | 100分 |
| **字体规范** | 字号、字重、行高、字体族（PingFang SC、Microsoft YaHei）符合设计文档 | 每发现1处不符合扣5分，扣完为止 | 100分 |
| **间距规范** | 使用8px网格系统，间距符合设计文档 | 每发现1处不符合扣3分，扣完为止 | 100分 |
| **布局结构** | 布局结构符合后台管理系统规范（顶部导航60px、侧边栏180px） | 布局结构不符合扣50分，部分不符合扣20-30分 | 100分 |
| **组件样式** | 按钮、卡片、表单等组件样式符合设计文档（Element Plus需定制样式） | 每发现1处不符合扣5分，扣完为止 | 100分 |
| **图标使用** | 图标风格、尺寸、颜色符合设计文档（Element Plus Icons、Lucide Vue Next） | 每发现1处不符合扣5分，扣完为止 | 100分 |
| **视觉一致性** | 同一页面内视觉元素保持一致 | 每发现1处不一致扣5分，扣完为止 | 100分 |

**评分计算**：
- **单项得分** = 100 - 扣分总和（最低0分）
- **视觉验收总分** = (各项得分之和) / 7
- **目标得分**：≥ 80分

**验收工具**：
- 设计文档对比（中车1.5前端设计文档.md）
- 浏览器开发者工具（检查CSS样式）
- 像素对比工具（Pixel Perfect）

- **单项得分** = 100 - 扣分总和（最低0分）
- **交互验收总分** = (各项得分之和) / 7

- 本项目不要求复杂的动效交互，但基本的过渡效果和状态反馈是必需的
- 所有交互反馈应使用Element Plus的消息提示组件（ElMessage）

## 九、综合评分计算

**最终得分计算公式**：

```
最终得分 = 视觉验收得分 × 25% + 
           交互验收得分 × 25% + 
           功能验收得分 × 20% + 
           性能验收得分 × 15% + 
           兼容性验收得分 × 10% + 
           代码质量验收得分 × 5% + 
           可访问性验收得分 × 5%
```

**验收标准**：
- **优秀**：最终得分 ≥ 90分
- **良好**：最终得分 ≥ 80分（目标）
- **合格**：最终得分 ≥ 70分
- **不合格**：最终得分 < 70分（需整改）

**验收流程**：
1. **开发自检**：开发人员按照验收标准自检，确保各项得分 ≥ 70分
2. **内部验收**：技术负责人验收，确保各项得分 ≥ 80分
3. **产品验收**：产品经理验收功能，确保功能验收得分 ≥ 80分
4. **最终验收**：所有验收通过，最终得分 ≥ 80分，页面标记为"验收通过"

## 十一、问题处理流程

**问题分级**：
- **严重问题**：扣分 ≥ 50分，必须修复后才能验收通过
- **一般问题**：扣分 20-49分，建议修复，影响最终得分
- **轻微问题**：扣分 < 20分，可选修复，不影响验收通过

**问题处理**：
1. **发现问题**：验收过程中记录所有问题，标注问题级别
2. **问题修复**：开发人员根据问题清单修复问题
3. **问题验证**：修复后重新验收，确认问题已解决
4. **验收通过**：所有严重问题修复，最终得分 ≥ 80分，验收通过

**验收记录**：
- 记录验收时间、验收人员、验收结果
- 记录发现的问题和修复情况
- 保存验收报告和评分明细

### B. 项目特定验收要点

#### B.1 技术栈验收

- **Vue 3.3.4**：必须使用Composition API和`<script setup>`语法
- **TypeScript 5.0.2**：支持JS/TS混合使用
- **Vite 4.4.5**：构建工具和开发服务器
- **Vue Router 4.2.4**：必须使用Hash模式（`createWebHashHistory`）
- **Element Plus 2.9.11**：UI组件库，需定制样式
- **Axios 1.9.0**：HTTP请求库，API_BASE_URL为`/exam`（开发环境）

#### B.2 项目结构验收

- API文件统一放在`src/api/`目录
- 组件放在`src/components/`目录
- 页面视图放在`src/views/`目录
- 工具函数放在`src/utils/`目录
- 类型定义放在`src/types/`目录

#### B.3 功能模块验收

- **考试管理**：自动组卷、手动组卷、组卷派发、审核流程
- **题库管理**：题库导入（Word格式）、题目类型（0-5）、题库审核
- **成绩管理**：成绩查询、导出、主观题评分、统计分析
- **用户管理**：教师管理、学生管理、批量导入（Excel格式）、角色权限（1-6）

#### B.4 API验收

- API基础URL：`/exam`（开发环境）或`/exam/api`（生产环境）
- 统一响应格式：`{ code: number, msg: string, data?: any }`
- 分页数据格式：`{ rows: [], total: number }` 或 `{ list: [], total: number }`
- 请求头包含：`Content-Type: application/json`、`Authorization: token`
